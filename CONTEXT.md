# Context Glossary

## Trusted Reflection Infrastructure

Working category for the human-first Reflections strategy.

Plain definition: software that helps people write privately, preserve personal context, and return to it as useful memory.

This is the primary strategic frame. It starts with people, thinking, writing, trust, memory, and understanding. AI enters after that foundation exists.

## Private AI Memory

Investor shorthand for the AI-enabled version of Reflections' long-term category.

Plain definition: software that helps users turn private, user-owned life context into useful memory through Invited AI.

Use this term after the human-first chain is established. The product is not defined by AI-first journal framing because the long-term claim is broader than journaling: personal context that becomes more valuable as it compounds.

Category alternatives to revisit later: personal reflection infrastructure, personal AI context layer, user-owned memory graph, personal memory infrastructure.

## Personal Memory OS

The long-term expansion vision for Reflections.

A Personal Memory OS helps a user preserve and revisit personal context across life domains: notes, moods, decisions, relationships, recurring patterns, future letters, and AI reflections. It is not a social feed, therapy service, productivity dashboard, or generic chatbot.

## Trusted Personal Context

Private information a user is willing to store because the product earns trust through behavior, not just copy.

Examples include emotional notes, relationship details, decisions, recurring thoughts, personal patterns, and memories. The trust requirement is high because this context is too intimate for many users to casually place in generic AI tools.

## Personal Memory Graph

The structured set of user-owned context Reflections can build from writing over time: people, decisions, moods, recurring themes, reminders, and Life Wiki entries.

The moat is not privacy alone. The advantage is a compounding user-owned personal memory graph created through trusted longitudinal interaction.

## Trusted Longitudinal Interaction

Repeated user interaction that compounds because the product keeps earning trust over time.

For Reflections, the causal chain is: trust enables honest writing, honest writing creates richer context, richer context creates better memory, and better memory should drive retention and paid demand.

## Invited AI

AI that acts when the user explicitly asks or enables a clearly bounded mode.

Invited AI is different from ambient AI. It should not feel like a watcher, background analyst, diagnosis engine, or constant coach. In Reflections, Invited AI supports reflection, Life Wiki refreshes, and pattern recognition without replacing writing as the center of the product.

## Encrypted Vault Mode

A user-configured mode that enforces full end-to-end device-level encryption for all journals, moods, and relationships. It disables all server-side and client-side AI analysis features (such as reflections, Life Wiki compilation, and relationship link cards) to guarantee total zero-knowledge privacy.

## Reflective Sanctuary Mode

A user-configured mode that stores notes in plaintext in the database (or server-side decrypted) to fully enable invited AI reflections, index summaries, and Life Wiki insights. This mode bypasses local key setups, passwords, and recovery phrase mechanisms for a simplified, AI-guided writing experience.

## Encrypted Personal Data Loop

The intended advantage loop for Reflections:

1. The product earns trust through private, low-pressure writing.
2. The user adds personal context they would not give to less trusted tools.
3. Encrypted storage protects that context.
4. Invited AI turns longitudinal writing into useful memory.
5. Better memory makes the product more valuable, which encourages continued writing.

This loop is only an investment-grade advantage if retention, conversion, and user interviews prove it.

## Investment Theorem

The single argument the pitch deck exists to prove.

For Reflections, the investment theorem is that private writing can become trusted long-term personal memory, and that memory can become the foundation for a larger personal intelligence platform.

## Assertion Headline

A slide title written as a claim the investor should believe after reading the slide.

Example: "Private writing is the trust mechanism for long-term personal memory." Avoid label headlines such as "Product," "Problem," or "Market."

## Active Assertion-Evidence Pair

A slide structure that pairs one assertion headline with one visual proof object.

Use it for the main pitch deck. Avoid topic headlines plus bullet lists unless bullets are the clearest proof.

## Evidence Slide

A slide that pairs one assertion headline with one visible proof object.

The proof object can be a metric, quote, screenshot, diagram, market assumption, comparison, or named experiment. A slide without evidence is either a thesis slide or a weak slide.

## Narrative Tension

The contrast between the current broken state and the better future Reflections makes possible.

Example: people write private thoughts to understand themselves, but that context is often scattered or forgotten. Reflections resolves that tension through private writing, trust, memory, and invited AI.

## Proof Obligation

The evidence a claim must eventually earn before it belongs in the main deck.

Claims without proof can appear as milestones or assumptions. They should not be presented as facts.

## Proof Status Label

A label attached to a major deck claim so the team does not overstate evidence.

Use three labels: Proven, Observed, and To be proven. Product screenshots can prove shipped capability, but they cannot prove user demand.

## Validation Roadmap

The replacement for a traction slide when user evidence does not exist yet.

For Reflections, the validation roadmap should show the 30-day evidence sprint and the 12-18 month milestone: proving memory-driven retention and conversion.

## Two-Sentence Layperson Test

A plain-language check for the company story.

The company should be explainable in exactly two simple sentences plus one concrete example before using investor category language.

## Squint Test

A visual hierarchy check for deck slides.

If the slide is blurred, the main structure should still be obvious: comparison, loop, timeline, scorecard, or product screenshot.

## Four-Chunk Limit

The main deck should show no more than four visible chunks of information at once.

Dense proof belongs in backup slides or a pre-read.

## Pre-Read

A document or appendix sent before or after the meeting for dense material.

For Reflections, pre-read candidates include market sizing, architecture, encryption flow, full competitor matrices, and raw user research.

## Audience-As-Hero

A framing rule for the pitch.

Reflections is the tool. The user is the actor who keeps ownership of private context and turns it into memory.
