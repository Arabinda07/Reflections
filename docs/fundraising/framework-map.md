# Pitch Deck Framework Map

## How Reflections will use the frameworks

The deck should prove a human-first investment case: people need a private place to think, writing creates trust, trust lets memory compound, and AI becomes useful after that context exists.

Frameworks are useful only when they clarify that argument. Skip anything that turns the deck into performance.

## Must use

| Framework | Role in the deck | How to use it |
| --- | --- | --- |
| Pyramid Principle / BLUF | Investor logic | Start with the answer: Reflections turns private writing into trusted long-term personal memory. Add AI only after writing, trust, and memory are established. |
| SCQA | Narrative setup | Situation: people need space to think. Complication: their private context is scattered or placed in tools that have not earned trust. Question: where will durable personal memory come from? Answer: from trusted reflection infrastructure. |
| So What | Slide discipline | Every slide must say what belief changes for the investor. If a slide only describes a feature, cut it or move it to backup. |
| YC seed framework | Early-stage clarity | Keep the deck short, plain, and specific about problem, insight, market, product, team, and proof plan. |
| Assertion-Evidence | Slide structure | Use claim headlines and pair each with one proof object: screenshot, diagram, market proxy, assumption map, or experiment. |
| Two-Sentence Layperson Test | Verbal clarity | Explain the company without category language first: private writing becomes memory the user can return to later. |
| Working Memory Chunking | Cognitive load | Keep main slides to four or fewer visible chunks. Move dense market, architecture, and encryption detail to backup. |

## Use selectively

| Framework | Role in the deck | Boundary |
| --- | --- | --- |
| Duarte Sparkline | Narrative tension | Contrast the current state of scattered private context with the better future of trusted personal memory. Keep it quiet, not theatrical. |
| Zuora narrative | Category framing | Use the world shift late enough that the user remains the protagonist. AI is part of the timing argument, not the opening hero. |
| Three Minute Thesis | Verbal constraint | Use it to simplify the founder pitch, not to design the whole deck. |
| Squint Test | Visual hierarchy | Use during visual design and slide review. A loop should look like a loop; a timeline should look like a timeline. |
| Ten-Minute Attention Reset | Live delivery | Use only for longer meetings, with a screenshot shift, short demo, or discussion pause. |
| Audience-As-Hero | Investor/user framing | Keep Reflections as the tool. The user is the actor who writes, remembers, and understands. |
| Asynchronous Pre-Read | Meeting design | Use for market sizing, architecture, encryption, metrics detail, and appendices. |

## Skip for now

| Framework | Why skip |
| --- | --- |
| Brad Feld board deck | This is for board governance, not an external pre-seed pitch. |
| Modular deck | Helpful later for investor variants, but premature before the core story is stable. |
| Steve Jobs keynote framework | Too theatrical for an early fundraising deck with limited traction. |
| Challenger Sale | Too B2B-enterprise-specific for the current consumer/pre-seed pitch. |
| 4MAT, IMRAD, Takahashi, SCIPAB | Good in their niches, but not the right organizing model for this investor deck. |

## Operating rule

Use the framework that removes confusion fastest:

1. Need the core answer? Use Pyramid Principle / BLUF.
2. Need the setup? Use SCQA.
3. Need to judge a slide? Use So What.
4. Need to write a slide? Use Assertion-Evidence.
5. Need plain language? Use the Two-Sentence Layperson Test.
6. Need emotional momentum? Use Duarte/Sparkline lightly.

Do not stack frameworks. Add another method only when it fixes a specific deck weakness.
