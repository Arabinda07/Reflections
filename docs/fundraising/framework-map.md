# Pitch Deck Framework Map

## How Reflections Will Use The Frameworks

The research ranks many presentation frameworks. Reflections only needs the few that improve investor belief without adding ceremony.

## Must Use

| Framework | Role in the deck | How to use it |
| --- | --- | --- |
| Pyramid Principle / BLUF | Investor logic | Start with the answer: Reflections is a Private AI Memory company, with private writing as the wedge. Support it with the smallest chain of claims needed. |
| SCQA | Narrative setup | Use situation, complication, question, answer to explain why now and why this company should exist. |
| So What | Slide discipline | Every slide must answer what the investor should believe or do differently after seeing it. |
| YC seed framework | Early-stage clarity | Keep the deck legible, short, jargon-free, and focused on problem, insight, market, product, team, and momentum. |
| Assertion-Evidence | Slide structure | Use claim headlines and pair each with one visual proof object. |
| Two-Sentence Layperson Test | Verbal clarity | Explain the company in two simple sentences plus one example before adding category language. |
| Working Memory Chunking | Cognitive load | Keep main slides to four or fewer visible chunks. |

## Use Selectively

| Framework | Role in the deck | Boundary |
| --- | --- | --- |
| Duarte Sparkline | Narrative tension | Use "what is" vs. "what could be" to create contrast, but do not make the deck theatrical. |
| Zuora narrative | Category framing | Use the global shift first: AI needs trusted personal context. Keep category language restrained until traction supports it. |
| Three Minute Thesis | Verbal constraint | Use it to simplify the founder pitch, not to design the whole deck. |
| Squint Test | Visual hierarchy | Use it during visual design and slide review, not as a strategy framework. |
| Ten-Minute Attention Reset | Live delivery | Use only for longer meetings, with a demo, screenshot shift, or discussion pause. |
| Audience-As-Hero | Investor/user framing | Use to keep Reflections as the tool and the user as the owner of the transformation. |
| Asynchronous Pre-Read | Meeting design | Use for market sizing, architecture, metrics detail, and appendices. |

## Skip For Now

| Framework | Why skip |
| --- | --- |
| Brad Feld board deck | This is for board governance, not an external pre-seed pitch. |
| Modular deck | Helpful later for investor variants, but premature before the core story is stable. |
| Steve Jobs keynote framework | Too theatrical for an early fundraising deck with limited traction. |
| Challenger Sale | Too B2B-enterprise-specific for the current consumer/pre-seed pitch. |
| 4MAT, IMRAD, Takahashi, SCIPAB | Good in their niches, but not the right organizing model for this investor deck. |

## Operating Rule

Use the framework that removes confusion fastest:

1. Need the core answer? Use Pyramid Principle / BLUF.
2. Need the setup? Use SCQA.
3. Need to judge a slide? Use So What.
4. Need to write a slide? Use Assertion-Evidence.
5. Need plain language? Use the Two-Sentence Layperson Test.
6. Need emotional momentum? Use Duarte/Sparkline lightly.

Skip framework stacking. Add another method only when it fixes a specific deck weakness.
